# Newifi D2 OpenWrt 固件编译问题总结

## 一、镜像超限问题

### 问题描述
编译完成后 `sysupgrade.bin` 无法生成，因为 squashfs 镜像超过了设备分区限制。

### 原因分析
```
设备分区限制: IMAGE_SIZE = 32448k (约 31.7MB)
之前失败时的镜像: vmlinuz(2.5MB) + root.squashfs(30.8MB) = 33.3MB
超出限制: 约 1.6MB
```

**超限的根本原因**：
1. 使用了 `luci-theme-argon` 和 `luci-theme-neobird`（体积大且不兼容）
2. 添加了 29 个中文语言包（每个约 10-50KB）
3. clash_meta 内核（9.5MB）被嵌入

### 解决方案：嵌入内核 + 精简其他组件

**目标**：在嵌入 clash 内核的前提下，控制镜像大小在 32448k 以内。

#### 体积估算
```
vmlinuz:                    ~2.5MB
root.squashfs (无内核):     ~21.5MB
clash_meta 内核 (压缩后):   ~6-7MB (在 squashfs 中)
────────────────────────────────────
预估总计:                   ~30MB ✅ 可控制在限制内
```

#### 具体措施

**1. 使用兼容主题（体积更小）**
```bash
# 使用 argon-mod（专为 coolsnowwolf lede 优化，体积更小）
CONFIG_PACKAGE_luci-theme-argon-mod=y

# 不要使用以下主题（体积大且不兼容）
# CONFIG_PACKAGE_luci-theme-argon is not set
# CONFIG_PACKAGE_luci-theme-neobird is not set
```

**2. 精简语言包（只保留必要的）**
```bash
# 必须保留（基础界面中文）
CONFIG_PACKAGE_luci-i18n-base-zh-cn=y
CONFIG_PACKAGE_luci-i18n-firewall-zh-cn=y

# 可选保留（常用功能中文）
CONFIG_PACKAGE_luci-i18n-opkg-zh-cn=y

# 移除其他语言包（节省约 500KB-1MB）
# 之前有 29 个语言包，精简到 3 个
```

**3. 嵌入 clash 内核**
```bash
# 在 AutoBuild_DiyScript.sh 中调用 ClashDL
ClashDL mipsle-softfloat meta
```

## 二、OpenClash 内核嵌入方案

### 内核下载与嵌入

**AutoBuild_DiyScript.sh 配置**：
```bash
case "${CONFIG_FILE}" in
d-team_newifi-d2*)
    # 下载并嵌入 clash meta 内核
    ClashDL mipsle-softfloat meta
    
    # 创建 uci-defaults 配置脚本
    BASE_FILES=${WORK_DIR}/openwrt/package/base-files/files
    mkdir -p ${BASE_FILES}/etc/uci-defaults
    cat > ${BASE_FILES}/etc/uci-defaults/99-openclash-settings << 'EOF'
#!/bin/sh
uci -q get openclash.config >/dev/null 2>&1 && {
    uci set openclash.config.core_version='linux-mipsle-softfloat'
    uci set openclash.config.release_branch='master'
    uci set openclash.config.proxy_mode='redir-host'
    uci commit openclash
}
exit 0
EOF
    chmod 755 ${BASE_FILES}/etc/uci-defaults/99-openclash-settings
    ;;
esac
```

### ClashDL 函数说明

`ClashDL` 函数定义在 `AutoBuild_Function.sh` 中：
```bash
# 用法
ClashDL <架构> <内核类型>

# 示例
ClashDL mipsle-softfloat meta   # 下载 clash_meta (mihomo) 内核
ClashDL mipsle-softfloat tun    # 下载 clash 原版内核
```

**架构参数**：
- `mipsle-softfloat` - Newifi D2 (MT7621) 使用
- `mipsle-hardfloat` - 部分 MT7621 设备使用

**内核类型**：
- `meta` - clash_meta (mihomo)，功能完整，约 9.5MB
- `tun` - clash 原版，功能较少，体积更小

### 内核存放位置

内核会被下载到：
```
/workdir/openwrt/package/base-files/files/etc/openclash/core/clash_meta
```

编译后会打包进 `base-files` ipk，最终嵌入到 squashfs 镜像中。

刷入路由器后，内核位于：
```
/etc/openclash/core/clash_meta
```

## 三、主题不兼容问题

### 问题描述
刷入固件后访问 LuCI 报错：
```
attempt to index global '__entries' (a nil value)
attempt to index field 'nodes' (a nil value)
```

### 原因分析

coolsnowwolf/lede 使用的是**旧版 luci 框架**（基于 OpenWrt 19.07/21.02 早期），而：

| 主题 | 设计目标 | 兼容性 | 体积 |
|------|----------|--------|------|
| `luci-theme-argon-mod` | coolsnowwolf lede 专用 | ✅ 兼容 | 较小 |
| `luci-theme-argon` (jerrykuku) | OpenWrt 21.02+/22.03 | ❌ 不兼容 | 较大 |
| `luci-theme-neobird` | OpenWrt 21.02+ | ❌ 不兼容 | 较大 |

**不兼容的具体表现**：
1. `dispatcher.node()` 函数依赖 `__entries` 全局变量，但旧版 luci 在模板渲染阶段此变量为 nil
2. 新版主题使用 `node.nodes` 访问子节点，但旧版使用 `node.children`
3. `node_childs()` 函数在旧版 dispatcher.lua 中不存在

### 解决方案

**使用兼容主题**：
```bash
# 在 .config 中配置
CONFIG_PACKAGE_luci-theme-argon-mod=y
CONFIG_PACKAGE_luci-theme-bootstrap=y  # 备用主题
```

**不要使用**：
- `luci-theme-argon` (jerrykuku 版)
- `luci-theme-neobird`

## 六、编译流程检查清单

### 编译前检查
- [ ] 配置文件使用 `luci-theme-argon-mod` 而非 `luci-theme-argon`
- [ ] 语言包只保留 3 个（base, firewall, opkg）
- [ ] AutoBuild_DiyScript.sh 调用 `ClashDL mipsle-softfloat meta`
- [ ] 确认 uci-defaults 脚本配置正确的 `core_version`

### 编译后验证
- [ ] 检查 `sysupgrade.bin` 是否生成
- [ ] 检查镜像大小是否 < 32448k (约 31.7MB)
- [ ] 检查 manifest 中包含 `luci-theme-argon-mod`
- [ ] 检查 base-files 中包含 `clash_meta` 内核

### 验证命令（在 VM 上执行）
```bash
# 检查镜像大小
ls -la /workdir/openwrt/bin/targets/ramips/mt7621/*sysupgrade.bin

# 检查 manifest
grep -E "argon-mod|openclash|clash" /workdir/openwrt/bin/targets/ramips/mt7621/*.manifest

# 检查内核是否嵌入
ls -la /workdir/openwrt/package/base-files/files/etc/openclash/core/
```

### 刷入后验证
- [ ] LuCI 页面能否正常访问（无 `__entries` 错误）
- [ ] 主题是否正常显示（argon-mod）
- [ ] OpenClash 内核是否存在：`ls /etc/openclash/core/clash_meta`
- [ ] OpenClash 能否正常启动

## 七、常见错误速查

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| `sysupgrade.bin` 未生成 | 镜像超限 | 精简语言包，使用 argon-mod |
| `__entries` is nil | 主题不兼容 | 使用 `luci-theme-argon-mod` |
| `nodes` is nil | 主题不兼容 | 使用 `luci-theme-argon-mod` |
| OpenClash 无法启动 | 内核缺失 | 检查 ClashDL 是否正确执行 |
| 页面空白/500错误 | dispatcher 问题 | 检查主题兼容性 |
